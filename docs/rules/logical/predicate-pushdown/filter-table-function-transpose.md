# Calcite FilterTableFunctionTransposeRule

## Description

Pushes a filter past a table function scan when the filter predicate
references only the pass-through columns of the table function. This
reduces the input to the table function, potentially reducing its
invocations.

**When to apply**: A filter sits above a table function, and the
table function has column mappings that identify pass-through columns.

**Why it works**: Pass-through columns are propagated unchanged from
the table function's input. Predicates on these columns can be
applied before the table function executes.

**Calcite class**: `org.apache.calcite.rel.rules.FilterTableFunctionTransposeRule`

## Relational Algebra

```algebra
-- Before: filter above table function
sigma[input_col > 5](TableFunc(R))
  where input_col maps to R's column

-- After: filter pushed below table function
TableFunc(sigma[R.col > 5](R))
```

## Preconditions


> **Note:** Formal preconditions are defined in the YAML frontmatter above.

```rust
fn applicable(
    filter: &Filter,
    table_func: &TableFunctionScan,
) -> bool {
    let mappings = table_func.column_mappings();
    if mappings.is_empty() {
        return false;
    }
    let pred_cols = filter.condition().referenced_columns();
    pred_cols.iter().all(|col| {
        mappings.iter().any(|m| m.output == *col)
    })
}
```

**Restrictions:**
- Table function must provide column mappings
- Only predicates on pass-through columns can be pushed
- Predicates on generated columns must stay above

## Cost Model

```rust
fn estimated_benefit(
    input_rows: f64,
    selectivity: f64,
    func_cost_per_row: f64,
) -> f64 {
    let saved_rows = input_rows * (1.0 - selectivity);
    saved_rows * func_cost_per_row
}
```

**Typical benefit**: 10-50% by reducing table function invocations.

## Test Cases

```sql
-- Positive: filter on pass-through column
SELECT * FROM TABLE(my_func(orders)) AS t
WHERE t.order_id > 1000;
-- order_id is passed through; push below function
```

```sql
-- Negative: filter on generated column
SELECT * FROM TABLE(my_func(orders)) AS t
WHERE t.computed_score > 0.5;
-- computed_score is generated by function; cannot push
```

```sql
-- Negative: no column mappings
SELECT * FROM TABLE(generate_series(1, 100)) AS t_result(x)
WHERE x > 50;
-- No column mappings available; cannot push
```
